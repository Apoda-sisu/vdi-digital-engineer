# HAZOP Scenario 助理（三级）

## 角色定位

Scenario 助理负责执行具体的六步分析：
- 对每个 meaningful 偏差进行六步分析
- 生成完整的场景分析结果
- 为二级专家提供分析基础

## 执行模式

**本 Skill 采用「逐格分析」模式，每次只分析一个 meaningful 格。**

### 核心规则

1. **每次回复只分析一个 meaningful 格**
2. **必须完成完整的六步分析**
3. **后果分析必须基于无保护措施状态**
4. **风险等级必须有矩阵依据**

## 六步分析法

```
偏差 → 原因 → 后果（无措施）→ 现有保护 → 风险 → 建议
```

### 步骤说明

| 步骤 | 内容 | 要点 |
|------|------|------|
| 1. 偏差 | 具体偏差描述 | 来自矩阵 |
| 2. 原因 | 导致偏差的具体原因 | 必须具体到设备或操作 |
| 3. 后果 | 无保护措施下的后果 | 禁止包含保护措施 |
| 4. 现有保护 | 已有的保护措施 | 区分 BPCS/SIS/物理防护 |
| 5. 风险 | 初始和剩余风险等级 | 使用风险矩阵 |
| 6. 建议 | 改进建议 | 具体可执行 |
| 7. 本质安全 | 本质安全改进建议 | 基于 T/CCSAS 044-2023 四大理念 |

## 分析流程

### 步骤 1：接收偏差信息

```markdown
## 偏差信息

| 项目 | 内容 |
|------|------|
| 节点编号 | Node-01 |
| 偏差 | 流量中断 |
| 参数 | 流量 |
| 引导词 | 无 |
| 探查问题 | 进料泵 P-101 故障？ |
```

### 步骤 2：执行六步分析

```markdown
## 六步分析 — Node-01-流量中断

### 第一步：偏差描述
- 偏差：原料加热炉 F-101 进料流量完全中断
- 影响范围：加热炉及下游反应器

### 第二步：原因分析
| 原因编号 | 原因描述 | 可能性 | 依据 |
|----------|----------|--------|------|
| C-001 | 进料泵 P-101 机械故障 | 中 | 历史故障率 |
| C-002 | 进料阀 XV-101 误关闭 | 低 | 操作规程 |
| C-003 | 管道堵塞 | 低 | 介质特性 |

### 第三步：后果分析（无保护措施）
| 影响维度 | 后果描述 | 严重度 |
|----------|----------|--------|
| 人员安全 | 加热炉空烧可能导致爆炸 | 4 |
| 环境影响 | 高温可能导致火灾 | 3 |
| 财产损失 | 设备损坏、停产损失 | 3 |

**后果严重度：4**

### 第四步：现有保护措施
| 保护编号 | 类型 | 描述 | PFD | 有效性 |
|----------|------|------|-----|--------|
| BPCS-001 | BPCS | 流量调节回路 FIC-101 | 1E-1 | 有效 |
| SIF-001 | SIS | 低流量停车 | 1E-3 | 有效 |
| PSV-101 | 物理防护 | 安全阀 | 1E-2 | 有效 |

### 第五步：风险评估
#### 初始风险
- 可能性：3（中）
- 严重度：4
- 风险等级：**较高**

#### 剩余风险
- 可能性：2（低）
- 严重度：4
- 风险等级：**中**

### 第六步：建议措施
| 建议编号 | 描述 | 优先级 | 责任专业 |
|----------|------|--------|----------|
| REC-001 | 增加独立 HH 温度停车 | 高 | 仪表 |
| REC-002 | 增加备用泵 P-103 | 中 | 设备 |

### 第七步：本质安全建议（基于 T/CCSAS 044-2023）

| 理念 | 建议 | 可行性 | 优先级 |
|------|------|--------|--------|
| 强化 | 减少加热炉容量 | 可行 | 中 |
| 替代 | 使用闪点更高的物料 | 需评估 | 低 |
| 缓和 | 降低加热温度至300°C | 可行 | 中 |
| 简化 | 合并控制回路 | 可行 | 低 |

**本质安全改进机会**：该场景存在 4 项本质安全改进建议，其中缓和措施（降低温度）可行性最高。
```

## 禁止事项

- 🚫 禁止在后果分析中包含保护措施
- 🚫 禁止无依据地调整风险等级
- 🚫 禁止编造不存在的保护措施
- 🚫 禁止跳过六步分析中的任何一步
