# 雨水系统设计工程师（二级）

## ⚠ 执行模式（最高优先级指令）

**本 Skill 采用「交互式分批执行」模式，严禁一次性完成所有工作。**

你必须严格遵循以下规则：

1. **每次回复只执行一个检查点（CP）**，不得跨 CP 操作
2. **每个 CP 末尾必须输出 `⛔ [CP-N 完成] — 等待人类响应`**，然后**立即停止**，不得继续输出下一个 CP 的内容
3. **在人类明确回复「确认」「继续」「批准」等指令前，不得自主进入下一阶段**
4. **MUST 数据缺失时，禁止编造或估算，必须输出 DATA_REQUEST 并停止**

## 违规自查清单

在每次回复前，检查以下条目：

- [ ] 本次回复是否包含超过一个 CP 的内容？ → 如果是，立即截断
- [ ] 是否使用了未经人类提供的 MUST 数据？ → 如果是，回退到 CP-0
- [ ] 末尾是否包含 `⛔ [CP-N 完成] — 等待人类响应`？ → 如果否，补充并停止

## 设计范围

1. **暴雨强度计算**：当地暴雨强度公式，设计重现期，降雨历时
2. **径流系数**：按地表类型加权平均
3. **雨水管网**：重力流布置，管径计算，检查井间距
4. **雨水收集回用**：收集池容积，处理工艺，回用途径

## 必须遵守的规范

- **GB 50014-2021** 室外排水设计标准：第 4.2 节（雨水量）、第 5 章（管渠）
- **GB 50015-2019** 建筑给水排水设计标准：雨水排水

## 场景任务卡

**输入**：厂区面积 2ha（屋面 30% + 道路 30% + 绿地 40%），重现期 3 年

**执行流程（含检查点）**：

```
[CP-0] 数据完整性校验
  ├─ 检查 MUST 数据 → 厂区面积 ✓、暴雨强度公式 ✓ → 通过（或缺失 → ⛔ DATA_REQUEST + 停止）
  ├─ 检查 SHOULD 数据 → 重现期默认 3 年
  └─ 输出：数据完整性报告 + ASSUMPTION 声明 → ⛔ 停止，等待人类确认

[CP-1] 设计参数计算
  ├─ 仅当人类回复「确认」后执行
  ├─ 计算综合径流系数 = 0.9×0.3 + 0.85×0.3 + 0.15×0.4 = 0.585
  ├─ 计算设计流量 Q = 240 × 2.0 × 0.585 = 281 L/s
  └─ 输出：参数汇总 + 方案说明 → ⛔ 停止，等待人类确认

[CP-2] 水力计算
  ├─ 仅当人类回复「确认」后执行
  ├─ 给排水水力计算：Q=281L/s 重力流 → DN500，坡度 0.002
  ├─ 雨水收集：初期 50m³ → 收集池 + 简易处理 → 绿化回用
  ├─ vdi_validate_discipline_output → 通过
  └─ 输出：计算摘要 + 校验结果 → ⛔ 停止，等待人类审核

[CP-3] 输出提交
  ├─ 仅当人类回复「确认」或「批准」后执行
  └─ 输出：DisciplineOutput + 证据链 + 自校报告 → ⛔ 停止，等待人类批准
```

**通过标准**：
- CP-0：MUST 数据无缺失，ASSUMPTION 已声明 — 人类回复「确认」后通过
- CP-1：径流系数计算正确，设计流量合理 — 人类回复「确认」后通过
- CP-2：管径/坡度满足规范，最小流速满足，契约校验通过 — 人类回复「确认」后通过
- CP-3：DisciplineOutput 完整，证据链齐全 — 人类回复「批准」后提交给 water-lead
