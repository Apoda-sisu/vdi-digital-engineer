## HAZOP 方法论（IEC 61882）

### 六步分析法

| 步骤 | 内容 | 输出 |
|------|------|------|
| 1. 定义范围 | 确定分析边界、目标、团队 | 分析计划 |
| 2. 节点划分 | 按功能/操作划分节点 | 节点清单 |
| 3. 偏差识别 | 引导词 × 参数 = 偏差 | 偏差矩阵 |
| 4. 原因分析 | 识别偏差原因及频率 | 原因清单 |
| 5. 后果评估 | 评估偏差后果及严重度 | 风险矩阵 |
| 6. 保护措施 | 识别现有保护措施，提出建议 | 建议措施表 |

### 引导词矩阵

| 引导词 | 含义 | 应用场景 |
|--------|------|----------|
| 无 (No) | 完全否定 | 流量中断、无进料 |
| 多 (More) | 量增加 | 流量过大、压力过高 |
| 少 (Less) | 量减少 | 流量不足、液位过低 |
| 反向 (Reverse) | 方向相反 | 逆流、反向传热 |
| 部分 (Part) | 不完整 | 组分缺失、相变 |
| 异常 (Other) | 非预期 | 杂质、副产物 |
| 早 (Early) | 时间提前 | 过早开启/关闭 |
| 晚 (Late) | 时间滞后 | 延迟开启/关闭 |

### 工艺参数偏差表

| 参数 | 典型偏差 | 常见原因 |
|------|----------|----------|
| 流量 | 无/多/少/反向 | 泵故障、阀门误操作、管道堵塞 |
| 压力 | 高/低/真空 | 泄放失效、管道破裂、泵扬程不足 |
| 温度 | 高/低 | 冷却失效、加热失控、环境温度 |
| 液位 | 高/低/空 | 进料异常、出料堵塞、仪表故障 |
| 组成 | 异常 | 原料变化、反应失控、分离失效 |
| 相态 | 变化 | 温度/压力变化导致相变 |

### 风险评估矩阵

| 严重度 \ 可能性 | 1-极低 | 2-低 | 3-中 | 4-高 | 5-极高 |
|----------------|--------|------|------|------|--------|
| A-轻微 | L | L | L | L | L |
| B-较轻 | L | L | L | L | M |
| C-中等 | L | L | M | M | H |
| D-严重 | L | M | M | H | H |
| E-灾难 | L | M | H | H | H |

- L (Low): 可接受，需监控
- M (Medium): 需采取措施降低风险
- H (High): 不可接受，必须立即采取措施

## 安全阀泄放场景速查

| 场景 | 泄放量计算基准 | 适用设备 |
|------|--------------|---------|
| 外部火灾 | API 521 §4.4.13.2（湿面积法） | 所有含液设备 |
| 出口阻塞 | 泵最大流量 or 压缩机最大能力 | 泵出口、压缩机出口 |
| 热膨胀 | 微量泄放（小喉径） | 充满液体的长管段 |
| 控制阀失效 | 全开 Cv × ΔP | 调节阀下游设备 |
| 冷却失效 | 回流中断 → 塔顶最大蒸发量 | 精馏塔 |

## HAZOP 角色整合

本 Skill 整合了原独立 HAZOP Skills 的功能，可承担以下角色：

### 1. HAZOP 主席能力

- 组织协调 HAZOP 分析全流程
- 引导讨论，确保方法论正确应用
- 审核各领域专家的分析结果
- 最终签发 HAZOP 报告

**执行模式**：阶段式审核，每个阶段需人类确认后方可进入下一阶段

### 2. HAZOP 工艺专家能力

- 基于工艺知识识别有意义偏差
- 分析偏差的工艺原因
- 评估工艺参数异常的影响
- 提供工艺改进建议

**执行模式**：逐节点分析，每次只分析一个节点

### 3. HAZOP 准备助理能力

- 节点划分和编号
- 设计意图整理
- 操作参数汇总
- 引导词矩阵应用

### 4. HAZOP 报告助理能力

- HAZOP 工作表生成
- 建议措施跟踪表
- 风险矩阵汇总
- 报告格式化输出

**执行流程（含检查点）**：

```
[CP-0] 数据完整性校验
  ├─ 调用 vdi_check_data_completeness(process, safety, {...})
  ├─ MUST 全部满足 ✓（或缺失 → ⛔ DATA_REQUEST + 停止）
  └─ 输出：数据完整性报告 + ASSUMPTION → ⛔ 停止，等待人类确认

[CP-1] HAZOP + SIL 定级
  ├─ 仅当人类回复「确认」后执行
  ├─ 从 P&ID 划分 HAZOP 节点（12 个）
  ├─ 偏差分析 → 保护措施 → 建议措施（标注负责人+状态）
  ├─ SIL 定级：LOPA 分析 → SIL 1/2/3
  └─ 输出：HAZOP 报告 + 建议措施跟踪表 + SIL 定级表 → ⛔ 停止，等待人类确认

[CP-2] 安全阀 + ESD + 防爆分区
  ├─ 仅当人类回复「确认」后执行
  ├─ 识别安全阀设备 → 泄放场景 → API 521 计算泄放量 → 喉径选型
  ├─ 汇总火炬总管泄放量
  ├─ 编制 ESD 因果矩阵（8-12 个联锁回路）
  ├─ 标注防爆分区（Zone 0/1/2）
  └─ 输出：安全阀选型表 + 火炬汇总 + ESD 矩阵 + 防爆分区图 → ⛔ 停止，等待人类确认

[CP-3] 输出提交
  ├─ 仅当人类回复「确认」或「批准」后执行
  └─ DisciplineOutput + 证据链 + 自校报告 → ⛔ 停止，等待人类批准
```

**通过标准**：
- CP-0：MUST 数据无缺失 — 人类回复「确认」后通过
- CP-1：HAZOP 节点 ≥ 10 个，建议措施跟踪表完整（每项有负责人+状态），SIL 定级有 LOPA 依据 — 人类回复「确认」后通过
- CP-2：安全阀泄放量 ≥ API 521 规范要求，ESD 矩阵 ≥ 8 条联锁，防爆分区标注完整 — 人类回复「确认」后通过
- CP-3：DisciplineOutput 完整，risk_level=high，citations ≥ 2 — 人类回复「批准」后提交
